### F-03 · `ReservationResponse` `LocalDateTime` timezone 불일치
- **위치**: `train-service/.../dto/ReservationResponse.java`
- **원인**: `reservedAt`, `expiresAt` 필드가 `LocalDateTime`(시간대 없음)으로 직렬화 → 서버 UTC, 브라우저 KST(UTC+9) → `new Date("2026-06-02T16:39:02")` → KST로 파싱 → 9시간 앞당겨짐 → `isExpired()` 즉시 `true` 반환
- **증상**: 예매 내역 빈 화면, 결제 페이지 타이머 즉시 00:00
- **수정**: `LocalDateTime` → `OffsetDateTime(UTC)` (`.atOffset(ZoneOffset.UTC)`)

### F-11 · `RefreshToken.rotate()` — 이전 access token blacklist 미지원
- **위치**: `auth-service/.../service/RefreshTokenService.java`
- **원인**: refresh token rotate 시 이전 access token을 blacklist에 추가하지 않음 → 탈취된 access token이 TTL(30분)까지 유효

## 🆕 2026-06-04 추가 발견·수정 (코드 직접 스캔 + 파생 추적)

### F-14 · `expireReservation` — detached 엔티티 미반영 + PAID race ✅ 수정
- **위치**: `train-service/.../service/ReservationService.java` `expireReservation`, 호출처 `ReservationExpiryScheduler`
- **원인**: 스케줄러가 트랜잭션 밖(`findByStatusAndExpiresAtBefore`)에서 로드한 **detached 엔티티**를 `@Transactional expireReservation`에 전달. detached 엔티티의 `reservation.cancel()`은 dirty-checking으로 flush되지 않음
- **증상**: status가 PENDING으로 남아 다음 쿼리에 또 잡힘 → **매 60초마다 rollback + `seat.released` 무한 반복**. 또한 쿼리~처리 사이 PAID로 바뀐 예약을 취소하는 race
- **수정**: 트랜잭션 내에서 `findById`로 managed 엔티티 재조회 + `status != PENDING` 가드(DB가 source of truth, T4)

### F-15 · `NotificationService.dispatch` — UNIQUE 위반 catch가 트랜잭션을 rollback-only로 마킹 ✅ 수정
- **위치**: `notification-service/.../service/NotificationService.java`
- **원인**: 단일 `@Transactional` 내 `save()`가 UNIQUE(correlation_id) 위반 시 `DataIntegrityViolationException`을 던지면 Hibernate가 트랜잭션을 rollback-only로 마킹. catch해도 커밋 시점에 `UnexpectedRollbackException` 발생 → Kafka 재시도 → DLT
- **증상**: N1 규칙(중복 이벤트 정상 멱등 처리)이 실제 DB에서 깨짐. 기존 단위 테스트는 Mockito mock이라 미검출
- **수정**: 신규 `NotificationLogWriter`(`@Transactional(REQUIRES_NEW)`)로 채널별 INSERT를 독립 트랜잭션 격리 → UNIQUE 위반이 해당 채널 트랜잭션만 롤백

### F-16 · `ReconciliationScheduler` — phantom 비트 제거가 in-flight 예약 비트 오삭제 ✅ 수정
- **위치**: `train-service/.../scheduler/ReconciliationScheduler.java` `removePhantomBits`
- **원인**: create()가 Lua 비트를 set한 뒤 트랜잭션 커밋 전(티켓 미가시)에 reconcile이 돌면, DB 미발견 → phantom으로 오인 → 비트 제거 + 허위 `seat.released` 발행
- **위험**: 서브초 윈도우의 더블부킹 가능성(커밋 후엔 DB 더블체크가 방어) + 알림 노이즈
- **수정**: 연속 2회 reconcile 모두 phantom일 때만 제거(`previousPhantomCandidates`) → in-flight 예약은 다음 사이클에 커밋되어 제외

### F-17 · `AuthService.refresh` — 비회원 refresh 불가 ✅ 수정
- **위치**: `auth-service/.../service/AuthService.java` `refresh`
- **원인**: 비회원도 `refresh` 토큰을 발급·저장받는데 `refresh()`는 `memberRepository.findById`만 조회 → 비회원은 `USER_NOT_FOUND`
- **수정**: Member 조회 실패 시 `NonMemberRepository` fallback으로 role/name 해소

### F-18 · `NonMemberService.register` — refresh 토큰 DB 미저장 (F-17 파생) ✅ 수정
- **위치**: `auth-service/.../service/NonMemberService.java` `register`
- **원인**: `register`는 refresh 토큰을 발급·반환하나 `login`과 달리 `refreshTokenService.save`를 호출하지 않음 → 등록 직후 rotate 시 `REFRESH_TOKEN_INVALID`
- **수정**: `refreshTokenService.save(userId, refresh, null, null, null)` 추가
